# Forcing 变量的参考高度

总结：Tair、RH、Uz的观测位置应该对应。若不对应，Uz应该进行转换。

## 问题

`forcing`（`Met` 结构体）中的气象驱动量，应输入冠层顶部还是 2m 位置？

## 模型物理结构

BEPS 的感热通量方程（以上层冠层为例）：

```
H_o = ρₐ * cp * (Tc_o - Tair) / ra_o
```

`ra_o` 由对数风廓线计算，描述从 **z_wind** 到冠层顶这段气柱的传输阻抗：

```julia
ustar = Uz * k / log((z_wind - d) / z0)   # d = 0.8 * h_canopy
ra_o  = 1/(k*ustar) * (log((z_wind-d)/z0) + 稳定度修正)
```

`Tair` 必须是 `z_wind` 处的温度，才能与 `ra_o` 配对构成物理一致的驱动梯度。

## 各变量要求

| 变量 | 参考高度 | 原因 |
|------|----------|------|
| `Uz` | `z_wind` 处 | 直接用于计算 `ustar` |
| `Tair` | `z_wind` 处 | `ra_o` 温度梯度的起点 |
| `RH` | `z_wind` 处 | VPD 驱动蒸腾的参考湿度 |
| `Rs`, `Rln_in` | 冠层顶以上 | 入射辐射（大气层内衰减可忽略） |
| `Prcp` | 冠层顶以上 | 截留模块从顶向下分配 |

`z_wind` 对应参数 `ps.veg.z_wind`，默认值 2m，需根据站点实际观测高度设置。

## 使用 2m Tair 的影响

### 情形一：低矮植被（h < 1.5m，z_wind ≈ 2m）

Uz、Tair、RH 均在 2m，完全一致，**无误差**。

### 情形二：高大森林 + 气象站数据（Tair@2m，Uz@10m，z_wind=10m）

`ra_o` 描述 10m → 冠层顶的阻力，但 `Tair` 取自 2m，驱动梯度起点不一致。
误差量级取决于 2m~10m 之间的温度廓线，通常较小。

### 情形三：高大森林 + 通量塔（z_wind=30m，Tair@2m）

最严重的情况：2m 处位于冠层内部，受冠层遮挡和土壤影响，与自由大气脱钩。

- `RH_2m` 林内偏高 → VPD 低估 → **蒸腾量系统性偏低**
- 晴天 `Tair_2m` 林荫下偏低 → `(Tc - Tair)` 高估 → **感热通量偏高**

## 实践建议

- **低矮植被**：2m 气象站数据可直接使用，设 `z_wind = 2`。
- **高大森林**：应使用冠层顶以上的观测值（通量塔塔顶），并将 `z_wind` 设为实际测量高度。
- **再分析数据（ERA5 等）**：Tair 为 2m，风速为 10m，二者高度不同，应注意：
  - 将 `z_wind` 设为风速观测高度（10m），
  - 或将 Uz 用对数廓线外推至 `z_wind`，再用对应高度的 Tair。

## 案例：z_canopy=15m，同时有 20m 和 2m 的观测

**结论：三个变量均用 20m，设 `z_wind = 20`。**

判断依据：

```
z_canopy = 15m  →  d = 12m,  z0 = 1.2m
对数廓线有效条件：z_wind > d + z0 = 13.2m

20m：> 13.2m  ✓  对数廓线成立，位于冠层上方自由大气
 2m：< 13.2m  ✗  位于冠层内部，廓线完全失效
```

| 变量 | 选用 | 原因 |
|------|------|------|
| `Uz` | 20m | 2m 在冠层内，对数廓线无效 |
| `Tair` | 20m | 与 Uz 保持一致；2m 受冠层遮挡，温度偏低（晴天）或偏高（夜间） |
| `RH` | 20m | 2m 林内湿度偏高，会系统性低估 VPD，导致蒸腾量偏低 |
| `ps.veg.z_wind` | 20 | 与观测高度一致 |

2m 的数据可用于林内微气候分析，但不应作为冠层能量平衡的大气驱动。

## 案例：z_tower=30m, z_canopy=15m, z_wind=10m, z_Tair=2m

这个情况有两个问题叠加：z_wind=10m < d = 0.8×15 = 12m，风速测量点在零面位移高度**以下**，对数风廓线直接失效（log 内为负数）。

```
z_canopy = 15m  →  d = 12m,  z0 = 1.2m
z_wind_measured = 10m < d = 12m  ← 对数廓线无效区间
z_Tair = 2m                       ← 林内
```

### Uz（最麻烦）

10m 处于冠层内部，风速遵循指数衰减廓线而非对数廓线。需要**两步外推**：

```
步骤1：冠层内部指数廓线 → 估算冠层顶风速 u(h)
  u(z) = u(h) * exp(-γ * (1 - z/h_canopy))
  γ ≈ 0.1 + LAI^0.75   （BEPS 内部公式，AeroConsts.jl:58）

  反解：u(h) = u(10m) / exp(-γ * (1 - 10/15))

步骤2：对数廓线从冠层顶外推到参考高度（如 30m）
  ustar = u(h) * k / log((h_canopy - d) / z0)
  u(30m) = ustar/k * log((30 - d) / z0)
```

然后令 `z_wind = 30`，`Uz = u(30m)`。**若塔顶（30m）有直接观测，直接用 `Uz@30m`，跳过外推。**

### Tair 和 RH（影响小）

干绝热递减率约 -6.5 K/km，2m→30m 仅差 **0.18°C**，可忽略：

```julia
Tair_corrected = Tair_2m + 6.5e-3 * (30 - 2)  # ≈ +0.18°C（可选）
```

RH 的修正更小，直接用 2m 值即可。

### 处理汇总

| 变量 | 操作 |
|------|------|
| `Uz` | 两步外推至 z_wind（或直接用塔顶观测） |
| `Tair` | 2m 值直接用，误差 < 0.2°C |
| `RH` | 2m 值直接用 |
| `ps.veg.z_wind` | 设为实际使用的 Uz 高度（建议 ≥ 20m） |
